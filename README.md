## теоретичнi вiдомостi щодо атаки часткового вгадування
Ця атака використовується, коли пряме розв'язання системи рівнянь, що описує шифр, є занадто складним (має занадто високу обчислювальну складність).

Основна ідея:
Замість того, щоб намагатися знайти всі невідомі змінні внутрішнього стану одночасно, криптоаналітик або зловмисник:

* Вгадує (фіксує) значення невеликої підмножини змінних внутрішнього стану.

* Визначає решту змінних, підставляючи вгадані значення в систему рівнянь шифру.

 * Перевіряє несуперечливість отриманого стану за допомогою відомих бітів гами.

## Атака на Bivium

Концепція:Bivium описується системою поліномів над K = GF(2). Оскільки повний стан Trivium (288 біт) вимагає вгадування понад 80 змінних (що перевищує складність brute-force ключа), аналіз фокусується на Bivium.

Механіка вгадування:
* Стан: 177 внутрішніх змінних
* Дані: 180–190 бітів гами (keystream) достатньо для формування повної системи рівнянь.
* Оптимізація: Метод Гаусса (Gaussian elimination) виділяє 22 головні (pivot) змінні та 36 вільних (free).
* Складність: Вгадавши 38 змінних (36 вільних + 2 додаткові), решта 61 визначається автоматично. Це знижує складність до $2^{38}$, що значно швидше за перебір ключа ($2^{80}$).

Результати:Найвищу ефективність показав алгоритм slimgb (Gröbner basis) — ~130 мс на ітерацію. SAT-солвери у цьому конкретному випадку працювали повільніше.

## Атака на KeeLoq
Концепція:
KeeLoq (стандарт для автосигналізацій) використовує 64-бітний ключ та 32-бітний блок. Атака поєднує вгадування частини ключа з алгебраїчним відновленням решти стану.

Ключові етапи:

* Вгадування: Достатньо зафіксувати лише 16 біт ключа.

* Алгебраїчний крок: Решта стану відновлюється через систему рівнянь (кубічні рівняння FSM та лінійні LFSR).

* Оптимізація (Fixed points): Пошук пар "відкритий текст/шифротекст", де стан змінюється мінімально, що різко спрощує систему поліномів.

Порівняння інструментів:
На відміну від Bivium, для KeeLoq SAT-солвери (MiniSat, CryptoMiniSat) виявилися вдвічі швидшими за базиси Грьобнера:
* SAT-solvers: 12–30 мс на крок.
* Gröbner basis: 27–180 мс на крок.

Підсумок: Повна атака (з урахуванням часу на пошук фіксованих точок) триває кілька годин.

## Атака часткового вгадування на A5/1 (GSM)
Концепція:Атака типу guess-and-determine на шифр A5/1 базується на вгадуванні стану найменшого регістра R_1 (19 біт) та подальшому обчисленні станів R_2 (22 біти) і R_3 (23 біти) через відому гаму (keystream).

Механіка атаки:
* Фіксація R_1: Перебираються всі $2^{19}$ варіантів значень першого регістра.
* Фільтрація через гаму: Оскільки $KS[i] = R_1[18] \oplus R_2[21] \oplus R_3[22]$, знання R_1[18] та KS[i] одразу відсіює 50% хибних комбінацій для R_2 та R_3.
* Поступове відновлення: Через правила нерегулярного тактування та наступні біти гами визначаються критичні біти (clocking bits $R_2[10], R_3[10]$ тощо).
* Відсікання (Pruning): Кандидати на внутрішній стан, що суперечать потоку гами, видаляються.

Чому це "часткове вгадування":Замість повного перебору 64-бітного стану, ми вгадуємо лише 19 біт, а решту 45 біт обчислюємо як залежні змінні через систему рівнянь шифру.

Часова складність: $2^{48.5}$ операцій (у $2^{15.5}$ разів швидше за brute-force)

# Базис для атаки часткового вгадування для шифру Strumok та приклад обчислення iнших значень внутрiшнього стану

AutoGuess показав, що для повного відновлення внутрішнього стану достатньо вгадати лише 7 змінних:
R2_3, S3, S19, S6, S7, R2_8, R2_10

Оскільки кожна змінна має розмір 64 біти, загальна теоретична складність такої атаки становить $2^{7 \times 64} = 2^{448}$, що є меншим за встановлений стандартом поріг стійкості $2^{512}$ для «Струмок-512».

За відомими значеннями Z0–Z10 та цими 7 змінними відновлюються всі 62 змінні стану, але для прикладу в реалізації ми відновили 13 зміних

Наприклад:

з R2_3 обчислюється R1_2; 

з R1_2 і S14 обчислюється R2_1;

з R2_1 обчислюється R1_0;

й так далі

Приклад кроку відновлення S₁₈: Використовується рівняння виходу: $Z_3 = S_{18} \oplus R1_3 \oplus R2_3 \oplus S_3$.Формула для обчислення:$$S_{18} = ((Z_3 \oplus S_3 \oplus R2_3) - R1_3) \pmod{2^{64}}$$
